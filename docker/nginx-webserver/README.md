# 나만의 웹 서버 띄우기 (nginx)

터미널(CMD 또는 PowerShell)을 열고 아래 명령어를 입력해 보세요.

## 1️⃣ 실제 인터넷 브라우저로 접속할 수 있는 웹 서버 띄우기
| 명령어 | 
| :--- | 
| docker run -d -p 8080:80 --name my-webserver nginx | 

🔍 **명령어 하나하나 뜯어보기 (중요!)** <br>

이 명령어가 도커의 실무적인 핵심입니다.

**-d (detached):** 컨테이너를 '백그라운드'에서 실행합니다. 이걸 안 쓰면 터미널 창에 로그가 계속 뜨면서 다른 명령어를 못 치게 돼요.

**-p 8080:80 (port):** **[내 컴퓨터 포트] : [컨테이너 안의 포트]** 를 연결합니다.

컨테이너 안의 웹 서버는 80번 문을 열고 기다리고 있어요. <br>
우리는 내 컴퓨터의 8080번 문으로 들어가면 컨테이너의 80번 문으로 연결되도록 통로를 뚫어준 겁니다.

--name my-webserver: 컨테이너에 내가 알아보기 쉬운 이름을 붙여준 겁니다. (이름이 없으면 도커가 마음대로 '재미있는 이름'을 붙여버려요.)

nginx: 사용할 이미지 이름입니다. (전 세계적으로 가장 많이 쓰는 웹 서버 소프트웨어예요.)

<hr>

🌐 **확인해보기** <br>

**브라우저 접속:** 인터넷 브라우저 주소창에 localhost:8080을 입력하고 엔터를 쳐보세요.
**"Welcome to nginx!"** 라는 문구가 보이면 성공입니다!

**Docker Desktop 확인:** 보내주신 캡처 화면 왼쪽에 Containers 탭을 눌러보세요.<br>
redis 외에 우리가 만든 my-webserver가 초록불이 들어온 채 돌아가고 있을 겁니다.

<hr>

🛠️ 실습 마무리: "흔적 지우기"
공부가 끝났으면 깔끔하게 치우는 법도 알아야겠죠? 기존에 있던 Redis는 건드리지 않고 우리가 만든 것만 지워봅시다.

* 컨테이너 멈추기:
  
| 명령어 | 
| :--- | 
| docker stop my-webserver |

* 컨테이너 삭제하기:
  
| 명령어 | 
| :--- | 
| docker rm my-webserver |

이제 다시 localhost:8080에 접속해 보세요. 아까와 달리 접속이 안 될 겁니다. 박스를 통째로 버렸기 때문이죠.
