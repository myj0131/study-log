# 도커(Docker)
이 폴더는 Docker 학습 과정에서 정리한 자료, 예제 코드, 프로젝트를 보관하는 공간입니다.  
기본 개념부터 실습 예제, 프로젝트 구성까지 단계적으로 학습한 내용을 체계적으로 정리합니다.
<hr>

## 1️⃣ 도커(Docker)란?
우리가 컴퓨터에 프로그램을 설치할 때를 생각해 보세요.<br>
게임 하나를 깔더라도 내 컴퓨터의 사양, 윈도우 버전, 그래픽 카드 드라이버 등이 다 맞아야 잘 돌아가죠?<br>
그런데 내 컴퓨터에서는 잘 되던 프로그램이 친구 컴퓨터나 회사 서버로 옮기면<br>
"어? 여기선 왜 안 되지?" 하는 상황이 자주 발생합니다. 설정이 다르거나 필요한 파일이 하나라도 빠졌기 때문이죠.

### 📦 '컨테이너' 비유
항구의 컨테이너 박스를 떠올려 보세요. 그 안에 옷이 들어있든, 냉장고가 들어있든, 배에 실어서 어디든 보낼 수 있습니다.<br>
배(운송 수단)만 있으면 내용물에 상관없이 목적지까지 안전하게 옮겨지고, 도착해서 문만 열면 바로 물건을 꺼낼 수 있죠.<br>

**도커**도 똑같습니다.
* 프로그램 코드 + 실행에 필요한 모든 환경(라이브러리, 설정 등)을 하나의 컨테이너 라는 박스에 통째로 집어넣습니다.
* 이 박스만 있으면 내 노트북, 회사 서버, 클라우드 어디서든 똑같이 실행됩니다. 
<hr>

## 2️⃣ 도커 장점
✅ **"내 컴에선 됐는데?" 방지**: 내 환경을 그대로 박스에 담았으니 어디서든 100% 동일하게 작동합니다.<br>

✅ **환경 설치의 간편함**: 예전에는 데이터베이스 하나 설치하려면 1시간씩 걸렸지만, 도커를 쓰면 명령어 한 줄로 1분 만에 끝납니다.<br>

✅ **격리성**: 컨테이너들은 서로 독립적입니다. A 프로그램이 고장 나도 B 프로그램에는 영향을 주지 않으며,<br>
&emsp;&emsp;&emsp;&emsp;&emsp;삭제하고 싶을 땐 박스만 버리면 내 컴퓨터는 깨끗한 상태를 유지합니다.
<hr>

## 3️⃣ 꼭 알아야할 3대 핵심 개념
| 용어 | 쉬운 설명 | 비유
| :--- | :--- | :--- |
| **이미지(Image)** | 컨테이너를 만들기 위한 **설계도** 또는 **압축파일** 입니다. | 붕어빵 틀 또는 게임 설치 CD
| **컨테이너(Container)** | 이미지를 실행시킨 **실체**입니다. 실제로 돌아가는 프로그램이죠. | 구워져 나온 붕어빵 또는 설치된 게임
| **도커 허브(Docker Hub)** | 남들이 미리 만들어 놓은 이미지를 공유하는 **저장소** 입니다. | **앱스토어** 또는 플레이스토어
| **도커 파일(Dockerfile)** | 이미지를 어떻게 만들지 순서대로 적어놓은 **레시피**입니다. | 붕어빵을 구울 때 **"반죽 먼저 넣고 팥 넣어라"** 고 적힌 메모지
| **볼륨(Volume)** | 컨테이너가 사라져도 유지되어야 하는 데이터를 담는 **외장 하드**입니다. | 붕어빵을 다 팔고 틀을 씻어도 **번 돈(데이터)** 은 따로 주머니에 보관하는 것
| **도커 허브(Docker Hub)** | 여러 컨테이너들이 서로 대화할 수 있게 연결해주는 **무전기**입니다. | 붕어빵 가게와 옆집 커피숍이 서로 주문을 주고받을 수 있는 **전화선**

<hr>

## 4️⃣ 개념 보충 설명
✅ **컨테이너는 '일회용'이다 (Persistence)**
* **현상:** 컨테이너 안에서 파일을 만들거나 일기를 써도, 컨테이너를 삭제하고 다시 실행하면 데이터가 싹 사라집니다.<br>
&emsp;&emsp;&nbsp;(공장 초기화 상태로 돌아가는 것과 비슷)<br>
* **해결책(Volume):** 그래서 진짜 중요한 데이터(데이터베이스 등)는 컨테이너 외부(내 컴퓨터 하드 디스크)에 따로 저장하도록 연력해 줘야 합니다.<br>
&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&nbsp;이걸 **볼륨(Volume)** 설정이라고 불러요.<br>

✅ **컨테이너는 '독립된 방'이다 (Port Forwarding)**
* **현상:** 컨테이너 안에서 웹 서버가 80번 포트에서 돌아가고 있어도, 밖(내 컴퓨터 브라우저)에서는 접속이 안 됩니다. 문이 잠겨 있거든요.<br>
&emsp;&emsp;&nbsp;(공장 초기화 상태로 돌아가는 것과 비슷)<br>
* **해결책(Volume):** 내 컴퓨터의 포트와 컨테이너의 포트를 연결해 주는 통로를 뚫어줘야 합니다.<br>
&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&nbsp;(예: 내 컴퓨터의 8080번으로 들어오면 컨테이너의 80번으로 보내줘!)<br>

✅ **설계도는 '레시피'로 만든다 (Dockerfile) :** 아까 이미지를 '설계도'라고 했죠? 이 설계도를 만드는 메모장이 바로 **Dockerfile**입니다.

* **"우분투 설치해줘"**

* **"파이썬 깔아줘"**

* **"내 소스 코드 복사해줘"**

* **프로그램 실행해줘" 이런 명령어를 순서대로 적어놓은 텍스트 파일입니다. 나중에 이 파일을 동료에게 주면 동료도 똑같은 이미지를 만들 수 있어요.**
<hr>

## 5️⃣ 도커는 어떻게 작용할까?(도커 흐름)
✅ **Build(만들기)** : 내 프로그램을 담은 이미지(설계도)를 만듭니다.<br>

✅ **Ship(보내기)** : 이미지를 저장소(도커 허브 등)에 올립니다.<br>

✅ **Run(실행하기)** : 어디서든 이미지를 내려받아 컨테이너(프로그램)을 실행합니다.
