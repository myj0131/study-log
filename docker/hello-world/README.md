# 도커(Docker) 설치 후 잘 설치 되었는지 확인 + 간단한 명령어 실행

## 1️⃣ 설치 확인
| 명령어 | 
| :--- | 
| docker --version | 

**결과:** Docker version 24.0.x ... 처럼 버전 정보가 나온다면 성공입니다!
<hr>

## 2️⃣ "Hello World" 실행하기
| 명령어 | 
| :--- | 
| docker run hello-world | 

**이 명령어 하나로 내부적으로는 엄청난 일들이 일어납니다. 화면에 출력되는 내용을 잘 보세요!** <br>

**무슨 일이 벌어진 걸까요? (핵심 과정)** <br>
**"Unable to find image... locally":** 도커가 먼저 내 컴퓨터에 hello-world라는 이미지(설계도)가 있는지 찾아봅니다. 없으니까 당황하지 않고 다음 단계로 넘어가죠. <br>

**"Pulling from library/hello-world":** 내 컴퓨터에 없으니, 인터넷(도커 허브)에서 설계도를 다운로드합니다. <br>

**"Hello from Docker!":** 다운로드한 설계도로 컨테이너를 만들어서 실행합니다. 이 메시지가 보인다면 컨테이너가 성공적으로 작동한 것입니다.
<hr>

## 3️⃣ 방금 만든 컨테이너 확인하기
| 명령어 | 
| :--- | 
| docker images | 

* 목록에 hello-world라는 이름이 보일 거예요. 이게 바로 우리가 다운로드한 **'이미지(설계도)'** 입니다. <br>

📅 CREATED 날짜의 진짜 의미
명령어를 입력했을 때 오늘 명령어를 입력한 날짜가 아닌 다른 날짜가 나올 겁니다. <br>
이 날짜는 이미지를 만든 사람이 설계도를 최종적으로 확정해서 세상에 내놓은 날짜입니다.<br>
도커 이미지는 일종의 **'통조림'**과 같습니다.

**제조일자:** 도커 관리자가 hello-world라는 프로그램을 다 만든 뒤, "자, 이제 완성!" 하고 박스에 포장한 날짜입니다. (이미지 상의 5개월 전)

**구입일자:** 당신이 오늘 docker run을 해서 인터넷에서 우리 집(내 컴퓨터)으로 가져온 날짜입니다.

도커 목록에서 보여주는 날짜는 **1번(제조일자)** 입니다. 그래서 오늘 처음 설치하고 오늘 처음 다운로드했더라도, <br>
그 이미지가 만들어진 지 좀 되었다면 5달 전, 1년 전으로 표시되는 것이죠.<br>

그다음, 실행했던 **컨테이너(실체)** 목록도 확인 해봅시다.

| 명령어 | 
| :--- | 
|  docker ps -a | 

hello-world 컨테이너가 생성되었다가 임무를 마치고 종료(Exited)된 상태인 것을 볼 수 있습니다.
